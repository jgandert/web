Glossary

  • Literal: Matches exact text characters.
  • Set: Matches any single character from a set or range.
  • Group: Groups tokens together, useful for capturing or quantifiers.
  • Named Group : A capturing group with a name, e.g. (?P<name>...) (Python-style).
  • Positive Lookahead (?=...) : Checks that what comes next matches, without consuming it. E.g. \d+(?= USD) matches 100 in "100 USD" but not in "100 EUR".
  • Negative Lookahead (?!...) : Checks that what comes next does not match. E.g. \d+(?! USD) matches 100 in "100 EUR" but not in "100 USD".
  • Positive Lookbehind (?<=...) : Checks that what comes before matches, without consuming it. E.g. (?<=\$)\d+ matches 50 in "$50" but not in "€50".
  • Negative Lookbehind (?<!...) : Checks that what comes before does not match. E.g. (?<!\$)\d+ matches 50 in "€50" but not in "$50".
  • Backreference : Matches the same text as previously matched by a group (e.g. \1 or \k<name>).
  • Branch (OR) : Matches the expression before or after the |.
  • Anchor : Matches a position like the start (^ ) or end ($) of a line.
  • Any (.): Matches any single character except newline.
  • Flags : Inline modifiers like (?i) for case-insensitive or (?x) for extended mode.
